STRENGTHS OF CONTENT-BASED FILTERING:
  + No cold-start problem for new users (just need liked products, not community data)
  + Fully explainable: "recommended because it's similar to X you liked"
  + Good for niche/long-tail products with little rating data
 
LIMITATIONS:
  - Filter bubble: if you only liked running shoes, you'll only see running shoes
  - Relies entirely on quality of product text descriptions
  - Cannot capture social signals (e.g. "trending right now")
  - Struggles with serendipitous discovery
 
HOW TO IMPROVE:
  1. Replace TF-IDF with sentence embeddings (sentence-transformers library)
     for semantic similarity instead of keyword overlap.
  2. Add collaborative filtering signals (what similar users liked).
  3. Use the hybrid blend (Model C above) to balance personalisation
     and broad popularity coverage.
  4. Increase field weights for title, decrease for boilerplate descriptions.
